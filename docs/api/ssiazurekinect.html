AzureKinect
  AzureKinect  SENSOR
 
           Azure Kinect DK Sensor. Before first use, make sure AzureKinect SensorSDK and BodyTrackingSDK are installed and added to the PATH.
 
             +  sr   DOUBLE   1   '30.00000'   LOCK   sample rate in hz (Kinect only supports 5.0, 15.0, and 30.0). Default: 30.0 
             +  rgbResolution   INT   1   '2'   LOCK   Resolution of the rgb video, default: 1080p. Set to OFF to deactivate. (0=2160p,1=1440p,2=1080p,3=720p,4=3072p,5=1536p,6=OFF). Option 3072p only up to sr = 15.0 
             +  depthMode   INT   1   '3'   LOCK   Depth mode, default: WFOV_BINNED. Set to OFF to deactivate (will be activated anyway with default if e.g. bodytracking or pointcloud are used). (0=NFOV_UNBINNED,1=NFOV_BINNED,2=WFOV_UNBINNED,3=WFOV_BINNED,4=PASSIVE_IR,5=OFF). Option WFOV_UNBINNED only up to sr = 15.0 
             +  nrOfBodiesToTrack   UINT   1   '0'   LOCK   The number of bodies to track. Default is 0, set to at least 1 to track someone! (Azure Kinect doesn't seem to have a limit for the nr of bodies it can track) 
             +  showBodyTracking   BOOL   1   'false'   LOCK   show body tracking, default: false (true, if at least one body is tracked). Only paints basic joints and bones, use the SkeletonPainter plugin for more options. 
             +  bodyTrackingSmoothingFactor   DOUBLE   1   '0.00000'   LOCK   Control bodytracking smoothing between 0.0 (none) - 1.0 (full). Default: 0.0. Less smoothing will increase the responsiveness of the detected skeletons but will cause more positional and orientational jitters. 
 
           Available channels:
 
             +  rgb   RGB image with resolution as set by options and 4 byte rgba pixels. 
             +  depthraw   Depth image with 2 byte depth-pixels signifying raw depth values in mm. 
             +  depthvisualisation   Depth visualisation with 3 byte rgb pixels ranging from red (near) to blue (far). 
             +  irraw   IR image with 2 byte raw values. 
             +  irvisualisation   IR image visualisation with 1 byte grayscale values. 
             +  skeleton   Reports 3D skeleton positions of 32 joints. 
             +  pointcloud   3-D coordinates (relative to Kinect camera) for every pixel in the depth image. 2 byte integer per dimension, unit: mm, positive Y points towards ground!
